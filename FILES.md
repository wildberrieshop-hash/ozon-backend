## 📋 Что находится в каждом файле?

### `main.py` — Основной бот (18 KB)

**Назначение:** Весь функционал Telegram-бота

**Содержит:**
- `OzonSupplyBot` класс — главный класс бота
- `SupplyForm` FSM — диалог создания поставки
- `SetupForm` FSM — диалог установки учетных данных
- Обработчики команд (`/start`, `/setup`, `/create`)
- Обработчики нажатий кнопок (колбеки)
- Методы интерфейса

**Когда читать:** Нужно добавить новую команду или изменить логику диалога

**Вход:**
```bash
python main.py
```

---

### `tgbot_db.py` — База данных пользователей (2.8 KB)

**Назначение:** Хранение и управление учетными данными пользователей Telegram

**Содержит:**
- Функция создания таблицы `tg_users`
- Функция сохранения учетных данных
- Функция получения учетных данных
- Функция проверки регистрации

**Когда читать:** Нужно модифицировать схему пользователей или добавить новые поля

**Пример использования:**
```python
import tgbot_db

# Инициализация БД
tgbot_db.init_users_table()

# Сохранение учетных данных
tgbot_db.save_user_credentials(
    telegram_id=123456789,
    username="john_doe",
    client_id="12345",
    api_key="abc_xyz_key"
)

# Получение учетных данных
creds = tgbot_db.get_user_credentials(123456789)
print(creds)  # {'client_id': '12345', 'api_key': 'abc_xyz_key'}
```

---

### `supply_flow.py` — Поток создания поставок (9.7 KB)

**Назначение:** Полная логика создания поставки в Озоне (от SKU до финальной поставки)

**Содержит:**
- `get_product_by_sku()` — поиск товара по артикулу
- `create_supply_full_flow()` — полный поток создания

**Когда читать:** Нужно добавить проверку, изменить логику валидации, или добавить новые шаги

**Пример использования:**
```python
import supply_flow
from ozon_client import OzonClient

client = OzonClient(client_id="...", api_key="...")

result = await supply_flow.create_supply_full_flow(
    client=client,
    sku="вак1",
    quantity=100,
    delivery_type="direct",
    cluster_ids=[1, 2, 3],
    target_date="2024-04-01"
)

print(result)
# {
#   'success': True,
#   'message': '✅ Поставка создана!...',
#   'order_id': 123456,
#   'draft_id': 999,
#   'cluster_info': {...}
# }
```

## 🚀 Типичные рабочие процессы

### Первый запуск

1. Читай: `QUICKSTART.md`
2. Копируй `.env.example` в `.env`
3. Добавь TELEGRAM_BOT_TOKEN
4. Запусти: `python main.py`

### Добавление новой команды

1. Изучи: `main.py` раздел "Регистрация обработчиков"
2. Добавь новый обработчик (функция + @self.dp.message/callback_query)
3. Протестируй в Telegram
4. Обнови `README.md` с описанием новой команды

### Решение проблемы при запуске

1. Читай: `SETUP.md` раздел "Решение проблем"
2. Проверь логи в консоли
3. Если не помогло — проверь `.env` файл

### Добавление шага в поток создания поставки

1. Изучи: `supply_flow.py` функция `create_supply_full_flow()`
2. Добавь новый шаг в нужное место
3. Добавь обработку ошибок (try/except)
4. Обнови документацию
