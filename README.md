# Telegram Bot для создания поставок Озон

Минималистичный бот для создания поставок на Озон прямо из Telegram.

## Возможности

- ✅ Сохранение учетных данных (Client-ID и API-Key)
- ✅ Выбор артикула (SKU)
- ✅ Выбор количества товара
- ✅ Выбор схемы доставки (прямая поставка или кросс-докинг)
- ✅ Выбор кластеров (галочки)
- ✅ Выбор даты поставки
- ✅ Подтверждение и создание поставки

## Установка

1. **Установи зависимости:**
```bash
cd TGBOT
pip install -r requirements.txt
```

2. **Создай файл .env на основе .env.example:**
```bash
# Windows
copy .env.example .env

# Linux/Mac
cp .env.example .env
```

3. **Получи Telegram Bot Token:**
   - Напиши боту `@BotFather` в Telegram
   - Выполни `/newbot`
   - Скопируй токен и вставь в .env:
```
TELEGRAM_BOT_TOKEN=123456789:ABCdefGHIjklmnoPQRstuvWXYZ1234567890
```

## Запуск

```bash
python main.py
```

Бот будет слушать на https://api.telegram.org и получать обновления (polling mode).

## Команды бота

### `/start`
Показывает приветствие и доступные команды. Если это первый раз, подскажет о необходимости `/setup`.

### `/setup`
Установка или обновление учетных данных:
1. Запрашивает Client-ID от Озона
2. Запрашивает API-Key от Озона
3. Сохраняет в локальной БД

### `/create`
Начинает процесс создания новой поставки:
1. **SKU** — Напиши артикул товара
2. **Количество** — Сколько единиц отправить
3. **Схема доставки** — Прямая поставка или кросс-докинг
4. **Кластеры** — Выбери галочками нужные кластеры
5. **Дата** — Выбери дату поставки
6. **Подтверждение** — Проверь всё и подтвердь

## Архитектура

- `main.py` — Основной бот, обработчики команд и FSM
- `tgbot_db.py` — Работа с БД пользователей
- `requirements.txt` — Зависимости проекта

## Безопасность

- ✅ Client-ID и API-Key хранятся в локальной SQLite БД
- ✅ Не отправляются на удаленные серверы
- ✅ Каждый пользователь имеет свои учетные данные
